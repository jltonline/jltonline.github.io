Minimales HTML + JS

Jede Seite wird als HTML mit möglichst wenig globalem Framework gepflegt:

<!DOCTYPE html>
<html>
  <head>
    <title>JLTonline</title>
    <meta charset="UTF-8">
    <script src="/js/jltonline.js" />
</head>
<body class="issue">
  <main>
    <!-- Inhalt -->
  </main>
</body>
</html>

Das JavaScript lädt die globalen Inhalte (Header/Footer/Sidebars, CSS, …) nach und kümmert sich um die Lokalisierung der HTML-Inhalte.

Der <main>-Teil einer Ausgabe könnte dann z.B. so aussehen:

<main class="issue">
  <h1>Bd. 12, Nr. 1 (2018)</h1>
  <h2>Narrative Unreliability: Scope and Limits</h2>

  <article>
    <p class="title">Table of Contents</p>
    <nav>
      <a href="contents-table-contents/958-5498-1-PB.pdf">PDF</a>
    </nav>
  </article>

  <h3>Schwerpunkt</h3>

  <article>
    <p class="title">Vorwort</p>
    <nav>
      <a href="aumuller-vorwort/946-5398-1-PB.html">HTML</a>
      <a href="aumuller-vorwort/946-5397-1-PB.pdf">PDF</a>
      <a href="aumuller-vorwort/946-5403-1-PB.html">VOLLTEXT</a>
    </nav>
    <p class="authors">Matthias  Aumüller</p>
  </article>

  <!-- ... -->
</main>

Lokalisierung könnte z.B. auf der Basis von lang-Attributen an den Elementen und ?lang={de,en} in der URL geschehen, das JS oder CSS kümmert sich darum, die Elemente in der “unpassenden” Sprache auszublenden.

Eine neue Ausgabe erfordert dann einen neuen Link in der Journalseite und eine neue Seite für die Ausgabe, letztere als Kopie von einer Vorlage oder der vorherigen Ausgabe.

Single-Page-App + Daten

Die Webseite wird als clientseitige Single-Page-App z.B. auf der Basis von Vue.js gebaut. Die Metadaten zu den Artikeln werden von der Redaktion in einer JSON- oder YAML-Datei gepflegt, z.B. (Mockup, JSON5):

[
  {
    vol: 12,
    number: 1,
    year: 2018,
    title: "Narrative Unreliability: Scope and Limits",
    sections: {
      toc: [{
          title: "Table of Contents",
          files: [
            {label: "PDF", path: "articles/12.1/toc.pdf"}
          ]
        },
      ],
      focus: [
        {
          title: "Unreliability and Narrator Types. On the Application Area of ›Unreliable Narration‹ [Abstract]",
          authors: "Janina Jacke",
          files: [
            {label: "HTML", lang: "en", path: "articles/12.1/jacke-abstract.html"},
            {label: "PDF", lang: "en", path: "articles/12.1/jacke-abstract.pdf"},
            {label: "FULL TEXT", lang: "en", path:  "articles/12.1/jacke-fulltext.html"},
            {label: "VOLLTEXT", lang: "de", path:  "articles/12.1/jacke-volltext.html"},
          ],
        },
        // ... weitere Artikel ...
      ],
      // ... weitere Abschnitte ...
    },
  },
  // weitere Ausgaben von _articles_
]

Die App lädt dann die JSON-Datei(en) und erzeugt die entsprechende Webseite bei jedem Aufruf im Browser der Endnutzer.

Generierungsansatz

Die Seiten werden z.B. in Markdown gepflegt und mit einem statischen Websitegenerator, z. B. auf der Basis von Pandoc, nach HTML übersetzt. Templates und Lokalisierung und Generierung von Zwischenseiten läuft dann über den Generator.

Lokalisierung

Im Prinzip kann ich mir drei Ansätze vorstellen:

Poor-Man’s Lokalisierung mit lang=
Es werden einfach alle Sprachvarianten in eine HTML-Datei geschrieben. Sprachspezifische Elemente werden mit Attributen (lang=“de”) gekennzeichnet und in der “anderen” Sprachversion durch das JS oder CSS entfernt. In der URL z.B. durch …?lang=de.
Templating/Übersetzungs-Bibliothek
Wir recherchieren eine Übersetzungsbibliothek (z.B.) für das verwendete Framework und benutzen die zur Produktion / Anpassung der sprechspezifischen Elemente.
Separate Sprachbäume
Pro Sprachversion ein eigener Dateibaum, in der URL entsprechend am Anfang.

HTML-Versionen der Texte

Die veröffentlichten Texte sind in einem speziellen HTML ohne Einbindung irgendeines Rahmens oder Templates. Denkbare Ansätze:

  • “Import”, d.h. Umbau in das auch sonst verwendete HTML-Format, und Anpassung des entsprechenden Konvertierskripts (kommen eh aus Word)
  • Laden durch ein … Lade-HTML/JS, mit komischen URLs (/html?path=/article/12.1/jacke-abstract&lang=de) oder URL-Rewriting.
